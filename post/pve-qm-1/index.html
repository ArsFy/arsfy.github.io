PVE (Proxmox VE) 提供 qm 命令管理所有 vm，本節將會展示如何使用 qm 命令獲取/變更 vm 的狀態。

獲取 VM 的狀態#

qm status <vmid>

VM 一共存在五種不同的狀態:

  • Running (運行中): status: running
  • Stopped (停止): status: stopped
  • Suspended (挂起): status: suspended
  • Unknown (未知): status: unknown
  • Not exist (不存在)
    Configuration file 'nodes/<host>/qemu-server/<vmid>.conf' does not exist

Golang 實例程式碼:

func GetVmStatus(id string) string {
	cmd := exec.Command("qm", "status", id)
	output, err := cmd.Output()
	if err != nil {
		log.Printf("Failed to get VM status: %v", err)
	}

	if !strings.Contains(string(output), "does not exist") {
		return strings.TrimSpace(strings.Replace(string(output), "status: ", "", -1))
	} else {
		return "not_exist"
	}
}

變更 VM 的狀態#

在執行部分動作時，作業系統可能不會回應你的請求，必要時請使用强制動作

qm xxx <vmid>

  • Start (啓動)#

    qm start <vmid>

  • Shutdown (關機)#

    這個動作可以選擇包含强制執行參數 --forceStop 1，作業系統可能不會回應這個動作

    qm shutdown <vmid>

  • Stop (停止)#

    這個動作可以選擇包含强制執行參數 --forceStop 1

    qm stop <vmid>

  • Reboot (重新啓動)#

    qm reboot <vmid>

  • Suspent (挂起)#

    保留當前 VM 狀態，並暫停 VM

    qm suspent <vmid>

  • Resume (恢復)#

    恢復被挂起的 VM

    qm resume <vmid>

  • Reset (重設)#

    强制重新啓動 VM，在 VM Clone -> Reset disk 后首次啓動引起的内核錯誤可以自動化執行這個動作解決

    qm reset <vmid>

  • Destroy (刪除)#

    刪除指定 VM 和指定 VM 相關的所有 Hardware (完整刪除)

    qm destroy <vmid>
